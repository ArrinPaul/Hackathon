# Plan: n8n Workflow Backend Support

## Goal
Add backend API endpoints and database tables to support the 3 n8n workflows described in `n8n-summary.md`: group registration, event ingestion with calendar fan-out, and scheduled reminders.

## New Database Tables

## New API Endpoints

### Workflow 1: `notifyme-group-register`
- **`POST /api/groups/register`**
  - Body: `{ telegram_chat_id, name }`
  - Upserts group record
  - Returns `{ group_id, invite_link }`
  - No auth required (called by n8n webhook)

### Workflow 2: `notifyme-ingestion`
- **`GET /api/groups/by-chat-id/:chatId`**
  - Returns `{ group_id, group_name }` for the given Telegram chat ID
  - No auth required

- **`POST /api/calendar/fan-out`**
  - Body: `{ event_id }`
  - Looks up event → group → all group_members
  - For each member with Google Calendar connected: creates calendar event using `google.js` service
  - Returns `{ fanned_out: N, failed: M }`
  - No auth required

### Workflow 3: `notifyme-reminders`
- **`GET /api/reminders/due`**
  - Query: none (finds all unsent reminders where event date is within the remind_offset_days window)
  - Returns array of reminder objects with event details + student telegram_username + group info
  - No auth required

- **`PATCH /api/reminders/mark-sent`**
  - Body: `{ reminder_id }` or query param `?remind_offset_days=N` + event context
  - Marks reminder as sent, sets sent_at
  - Returns `{ success: true }`
  - No auth required
