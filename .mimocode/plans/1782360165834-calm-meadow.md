# CampusFlow Implementation Plan

## Scope
Create the full frontend (Next.js 14) and backend (Express.js) for:
1. **Landing page** (`/`)
2. **Sign In page** (`/login`)
3. **Sign Up page** (`/signup`)
4. **Dashboard** (`/dashboard`) with sidebar, widgets, and navigation

Strictly following `Documents/DESIGN_PLAN.md` for all design tokens, colors, layouts, and component patterns.

---

## Phase 1: Project Scaffolding

### 1.1 Initialize Next.js Frontend
- Run `npx create-next-app@latest frontend --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` inside `D:\Project`
- Install additional deps: `class-variance-authority clsx tailwind-merge tailwindcss-animate framer-motion lucide-react`
- Install Shadcn UI: `npx shadcn@latest init` (set up `components.json`, `src/lib/utils.ts` with `cn()` helper)
- Add Shadcn components: `button card input label badge separator`

### 1.2 Initialize Express Backend
- `mkdir backend && cd backend && npm init -y`
- Install deps: `express cors dotenv jsonwebtoken bcryptjs groq-sdk axios @supabase/supabase-js`
- Create folder structure: `src/routes/`, `src/services/`, `src/middleware/`

### 1.3 Design Tokens & Global Styles
**File:** `frontend/src/app/globals.css`
- All CSS variables from DESIGN_PLAN.md (light + dark mode)
- Custom scrollbar styles (`.sidebar-scroll`)
- `.transition-standard` utility class

**File:** `frontend/tailwind.config.ts`

## Phase 7: Backend API

### 7.1 Express Server
**File:** `backend/src/index.js`
- Express + CORS + JSON body parser
- Routes: `/api/auth`, `/api/tasks`, `/api/ai`, `/api/notices`, `/api/attendance`, `/api/automations`
- Listen on PORT (default 4000)

### 7.2 Auth Routes
**File:** `backend/src/routes/auth.js`
- `POST /register` — hash password with bcryptjs, insert into `students` table, return JWT
- `POST /login` — find by email, verify password, return JWT
- `GET /me` — verify JWT middleware, return user profile

### 7.3 Tasks Routes
**File:** `backend/src/routes/tasks.js`
- `GET /` — list student's tasks (sorted by deadline)
- `GET /today` — tasks where deadline = today
- `GET /upcoming` — tasks where deadline within 7 days
- `POST /` — create task
- `PUT /:id` — update task
- `DELETE /:id` — delete task

### 7.4 AI Routes
**File:** `backend/src/routes/ai.js`
- `GET /tip` — return random study tip (hardcoded for MVP, Groq in V2)
- `POST /summarize` — Groq API call for notice summarization
- `POST /attendance-alert` — calculate attendance risk

### 7.5 Auth Middleware
**File:** `backend/src/middleware/auth.js`
- Extract JWT from `Authorization: Bearer` header
- Verify with jsonwebtoken
- Attach `req.student` from decoded token

### 7.6 Services
| File | Purpose |
|------|---------|
| `backend/src/services/supabase.js` | Supabase admin client (service role key) |
| `backend/src/services/groq.js` | Groq AI wrapper functions |
| `backend/src/services/n8n.js` | n8n webhook caller functions |

### 7.7 SQL Schema
**File:** `sql/schema.sql`
- All 5 tables from TRD.md (students, tasks, notices, attendance, automation_logs)
- Plus preferences table from DESIGN_PLAN.md

### 7.8 Environment Files
**File:** `backend/.env.example`
- All required env vars with placeholders

**File:** `frontend/.env.local`
- NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, NEXT_PUBLIC_BACKEND_URL
